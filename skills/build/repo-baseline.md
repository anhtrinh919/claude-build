### CHANGELOG.md format

Keep a Changelog convention (keepachangelog.com). Auto-generated by `/build-spec` (replan mode).

```markdown
# Changelog

## [Unreleased]

## [1.0.0] - YYYY-MM-DD
### Added
- ...
### Changed
- ...
### Fixed
- ...
```

Semantic versioning: Phase 0 â†’ `0.1.0` (clickable but not functional). Phase 1 (first working feature) â†’ `1.0.0`. Each slice increments MINOR. Hotfixes increment PATCH.

### .gitattributes

Normalizes line endings and marks binary files for clean cross-platform diffs.


### .editorconfig

Consistent code style across editors without requiring a specific IDE.

```ini

[*]

[*.md]

[Makefile]
```


### Branch strategy

- Default: `main` (never `master`)
- Feature: `feature/short-description` (kebab-case, â‰¤5 words); fix: `fix/...`; chore: `chore/...`
- SDD pipeline uses `phase-N-[feature-slug]` â€” these coexist

Branch protection on `main`: require PR reviews if team project; optional for solo.

### Commit message convention

Format: `type: description` (lowercase, present tense, â‰¤72 chars)

Types: `feat` (new feature), `fix` (bug fix), `chore` (maintenance), `docs` (documentation), `test` (tests), `refactor` (restructure without behavior change), `style` (formatting)

Examples:
- `feat: add user profile settings page`
- `fix: correct CORS origin for production domain`
- `chore: update dependencies`

### LICENSE

Default: no LICENSE file (absence = all rights reserved). If user mentioned open source during `/build-spec` (constitution mode), generate MIT with current year.
