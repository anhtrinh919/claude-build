# Auto-continue — one turn per phase, stop only at named gates

A phase runs start-to-finish in ONE turn. Internal step and milestone boundaries are NOT stopping points — cross them silently and keep working.

**The ONLY places you stop and yield the turn (the user gates):**
- the **constitution** boundary (approve the product story),
- each **phase-complete** boundary (approve the dogfood, go/no-go the next phase),
- the **deploy** boundary (go/no-go into Milestone 3),
- a genuine **felt-impact fork** (a decision the user will feel, no strictly-better option — `voice.md`),
- a **cap-hit binary** ("Accept anyway, or Stop?"),
- the **external design hand-off** — the brief is delivered and the user goes away to design in their own tool; nothing can proceed until they return with exported images.

**Everywhere else — do NOT:** end the turn on a step/milestone boundary · say "next I'll…" / "I'll do X next" · ask permission to proceed · narrate a pause · frame a sub-skill return as a stopping point. A sub-skill returning hands straight back to the orchestrator, which runs its gate and continues in the SAME turn.

**Showing the user something is not stopping.** A rendered mockup, a review finding, a passing check — show it and keep going. Only the named gates above wait for an answer.
